Accessibility statement.

Last updated · [TBD] · Reviewed every [TBD] months
Plain English: we want every contractor, client, and visitor to be able to use SalesFlow regardless of device, ability, or context. We fall short in places. This page is honest about where.

01Our commitment.

Accessibility is not a checkbox we hit once a year. We test with assistive technology, listen to feedback from contractors, and fix what's broken.

02Standard we follow.

We aim for WCAG 2.2 AA across the web app and marketing site. The mobile contractor app follows platform accessibility guidelines (iOS / Android).

03What works today.

  • Full keyboard navigation on all primary flows.
  • Screen-reader labels on every interactive element in the contractor app.
  • Respects user's OS font-size and contrast preferences.
  • Dark and light modes with [TBD]:1 minimum text contrast.
  • Captions on all tutorial videos.

04Known issues.

  • The map view on the contractor app is not fully screen-reader accessible. A list-view alternative is always available alongside.
  • Some demo sites we generate for clients inherit accessibility limitations from client-provided assets. We flag these for the orchestration desk.

05Feedback.

If something's broken for you, tell us. Email accessibility@salesflow.[TBD] and we'll reply within [TBD] working days with a fix or a plan. Your feedback makes this better for everyone.
